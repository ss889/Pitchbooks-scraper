# Sprints Directory

This directory contains all sprint documentation organized by status.

## Structure

```
sprints/
├── planning/     # Sprints being planned (requirements, design)
├── active/       # Currently executing sprints
└── completed/    # Finished and validated sprints
```

## Sprint Lifecycle

1. **Planning** → Requirements gathering, test specs, task breakdown
2. **Active** → TDD implementation cycles (Red-Green-Refactor)
3. **Completed** → Validated, tested, documented
